# Frontend Design Brief — Phase 5–7 Remediation (2026-02-16)

This brief records the required `$frontend-design` workflow for the remaining pipeline UI parity fixes.

## Surface A — Verify Actionable Links

- Purpose/User Goal: After verification, operators need immediate jump points to the exact reviewed cards and cited evidence.
- Aesthetic Direction: Operator dashboard clarity with compact, high-signal pills and link chips.
- Memorable Differentiator: “Action strip” links that turn verify output from passive status text into direct evidence navigation.

## Surface B — Go Live Provider/Model Visibility

- Purpose/User Goal: Show exactly what model/adapters are active before live usage, with one-click path to configuration details.
- Aesthetic Direction: Command-deck style handoff card with explicit model identity and adapter roster.
- Memorable Differentiator: Go Live card now includes an inline provider/model configuration aperture, not hidden diagnostics.

## Surface C — Rollback: Restore Last Published

- Purpose/User Goal: Recover safely from a bad publish by restoring previous published pointers without CLI steps.
- Aesthetic Direction: High-trust safety affordance in the wizard header.
- Memorable Differentiator: One-click rollback action with explicit restored store/episode pointer feedback.

## Surface D — Atom Conflict Marking

- Purpose/User Goal: Let operators mark conflicts from memory detail without leaving the current inspection context.
- Aesthetic Direction: Embedded corrective control inside detail pane, preserving flow.
- Memorable Differentiator: Conflict-marking mini-form colocated with provenance/citation context for rapid trust corrections.
